Типовые конфигурации
1С:ERP Управление предприятием 2, 1С:Комплексная автоматизация 2

Руководство по переходу с "1С:Управление производственным предприятием", ред.1.3 / "1С:Комплексная автоматизация", ред. 1.1

Данный документ является руководством для пользователей по переходу с типовых конфигураций "1С:Управление производственным предприятием", редакция 1.3 / "1С:Комплексная автоматизация", редакция 1.1, на типовые конфигурации "1С:ERP Управление предприятием 2" / "1С:Комплексная автоматизация 2".

Используемые в документации сокращения:

  • УПП 1.3 – типовая конфигурация "1С:Управление производственным предприятием", редакции 1.3.
  • КА 1.1 – типовая конфигурация "1С:Комплексная автоматизация", редакции 1.1.
  • УП 2 – типовая конфигурация "1С:ERP Управление предприятием 2".
  • КА 2 – типовая конфигурация "1С:Комплексная автоматизация 2".
  • РЦ - рабочий центр.
  • ПВД - правило выгрузки данных.
  • ТЧ - табличная часть.
  • РАУЗ - учетная политика "Расширенная аналитика учета затрат".
  • НСИ - нормативно-справочная информация.

В целях упрощения в тексте руководства используются следующие обобщения: под конфигурацией-источником следует понимать УПП 1.3 / КА 1.1, а под конфигурацией-приемником - УП 2 / КА 2 (в зависимости от вида выполняемого перехода).

Порядок переноса данных

Подготовка информационных баз к переносу данных

Перед запуском переноса необходимо подготовить информационную базу-источник. Подготовка выполняется в несколько шагов.

Шаг 1. Резервное копирование информационной базы

Необходимо выполнить резервное копирование информационной база, из которой выполняется перенос данных.

Для выполнения резервного копирования необходимо войти в конфигуратор, выбрать пункт меню Администрирование, далее Выгрузить информационную базу. В появившемся диалогом окне указать имя файла архивной копии и нажать на кнопку Save As (Сохранить как).

Шаг 2. Обновление версии конфигурации

При запуске в информационной базе-приемнике помощника перехода отображается минимальная версия (релиз) конфигурации-источника, с которой поддерживается перенос. Если версия конфигурации-источника ниже указанной в обработке, необходимо провести обновление конфигурации.

Шаг 3. Удаление помеченных объектов

Перед началом переноса данных в информационной базе-источнике необходимо выполнить процедуру удаления помеченные на удаление объектов (пункт меню Операции > Удаление помеченных объектов > Контроль > Удалить).

Шаг 4. Подготовка данных

Для корректного переноса данных необходимо провести ряд регламентных операций по подготовке данных и закрытию месяца.

Необходимо запустить обработки: восстановление состояния расчетов с контрагентами, проведение по регистрам НДС, проведение по партиям в случае, если ведется партионный учет, 

Также необходимо выполнить другие регламентные операции по закрытию месяца. Для этого рекомендуется использовать бизнес-процесс Закрытие месяца:

  • начисление амортизации основных средств и нематериальных активов;
  • погашение стоимости спецодежды и спец.оснастки в эксплуатации;
  • переоценка валютные средства;
  • списание расходов будущих периодов;
  • списание прямых и косвенных расходов на результаты хозяйственной деятельности;
  • определение финансового результата и закрытие счетов 90 и 91;
  • произведение расчетов по налогу на прибыль.

После выполнения указанных операций также не должно быть отрицательных количественных или суммовых остатков по товарам на складах. Такие остатки не будут перенесены.

Выгрузка данных для перехода

Запуск обработки выгрузки

Для создания файла выгрузки данных необходимо в информационной базе-источнике запустить обработку Выгрузка данных.epf. Указанная обработка входит в комплект поставки конфигурации УП 2 / КА 2.

Выбор конфигурации для перехода

На первом шаге работы с обработкой Выгрузка данных.epf необходимо выбрать из списка конфигурацию, на которую осуществляется переход.

Выбор организаций и даты остатков

На следующем шаге необходимо указать дату остатков, на которую будут сформированы документы ввода остатков и перечень переносимых организаций.

Выбор основных правил переноса нормативно-справочной информации и остатков

На следующем шаге обработка перехода предлагает выбрать основные виды данных для переноса:

  • настройка параметров учета,
  • нормативно-справочная информация,
  • начальные остатки.

На данном шаге необходимо отметить флажками необходимые виды данных, и нажать Далее.

Перенос зарплатных данных и данных кадрового учета

На этом шаге необходимо отметить флажками те виды данных по зарплате и кадровому учету, которые должны быть перенесены в информационную базу-приемник. Методика переноса данных зарплаты и кадрового учета совпадает с методикой переноса данных при переходе из редакции 2.5 в редакцию 3.0 конфигурации "1С:Зарплата и управление персоналом". Описание методики содержится в статье на сайте ИТС.
Следует отметить, что для переноса используется рекомендуемый ("упрощенный") вариант перехода, т.е. первичные кадровые документы не переносятся, а формируются документы ввода остатков "Начальная штатная расстановка", "Начальная задолженность по зарплате" и т.д.

Выгрузка данных

На последнем шаге необходимо выбрать файл, в который будут выгружены данные.
Этот файл потребуется указать в помощнике перехода базы-приемника.
По нажатию кнопки Выгрузить данные запускается процесс переноса данных.
После окончания выгрузки работу с базой-источником можно завершить. Дальнейшая работа будет вестись в базе-приемнике.

Помощник перехода

Запуск помощника

Для запуска помощника перехода в информационной базе-приемнике необходимо перейти в раздел Администрирование > Начальное заполнение, далее выбрать соответствующую команду.

Выбор источника и загрузка данных

На втором шаге работы помощника необходимо указать путь к файлу выгрузки данных, полученному на предыдущем этапе, и нажать кнопку "Загрузить данные".
На этом процесс переноса данных завершен.

Перенос нормативно-справочной информации

Производственный учет

Группы заменяемости рабочих центров => Виды рабочих центров
Группы заменяемости рабочих центров в УПП 1.3 и КА 1.1 могут содержать рабочие центры (РЦ), принадлежащие различным подразделениям. Таким образом, одна группа заменяемости РЦ может переноситься в несколько видов РЦ информационной базы-приемника, которые будут принадлежать различным подразделениям.

Рабочие центры => Рабочие центры
Взаимосвязь рабочего центра с подразделением в УП 2 и КА 2 устанавливается через Вид рабочего центра (обязательный для заполнения реквизит справочника Рабочие центры).

Если рабочий центр в информационной базе-источнике не принадлежит ни одной группе заменяемости, то при переносе в информационной базе-приемнике для него создается новый вид рабочего центра с соответствующим наименованием. Таким образом, рабочие центры, не состоящие ни в одной группе заменяемости, переносятся в отдельные виды рабочего центра.

Если же в составе группы заменяемости присутствуют рабочие центры разных подразделений, то при переносе к исходному наименованию группы заменяемости добавляется наименование подразделения. Таким образом одна группа заменяемости может переноситься в несколько видов рабочих центров.

Спецификации номенклатуры => Ресурсные спецификации
В справочник Ресурсные спецификации переносятся только табличные части Исходные комплектующие и Выходные изделия.

Спецификация номенклатуры в может содержать в списке комплектующих номенклатурные узлы или сборочные спецификации. При переносе для узлов или сборочных спецификаций создаются ресурсные спецификации. Они содержат единственное выходное изделие, наименование которого берется из названия номенклатурного узла. Вместо номенклатурного узла в ресурсную спецификацию заполняется указанное выходное изделие, также включается флажок Производится в процессе и заполняется значение ресурсной спецификации.

Номенклатурные узлы => Ресурсные спецификации
При переносе создается элемент справочника Номенклатура, наименование которого заполняется из названия номенклатурного узла. Создается ресурсная спецификация, эта номенклатура добавляется в табличную часть Выходные изделия.

Материалы из состава узла добавляются в табличную часть Исходные материалы.

Бюджетирование (только для перехода из УПП1.3 в УП2)

Справочник Бюджетные отчеты из УПП 1.3 переносится в УП 2 в справочник Виды бюджетов. Виды бюджетов привязываются к модели бюджетирования с наименованием "Основная модель". Если данная модель не найдена в УП 2, она создается.

Для видов бюджетов создается структура в виде таблицы. В колонках заполняется измерение - Месяц, в строках - Статьи бюджетов.

Справочник Статьи оборотов по бюджетам переносится в справочник Статьи бюджетов.

Счета плана счетов бюджетирования переносятся в значения справочника Показатели бюджетов.

Связи статей бюджетов и плана счетов бюджетирования в УПП 1.3 находятся в табличной части Проводки справочника Статьи бюджетов и переносятся в регистр Связи статей и показателей.

После завершения переноса данных необходимо в пользовательском режиме настроить правила получения фактических данных.
Таким образом без дополнительных настроек после переноса возможно создавать документы Экземпляр бюджетного отчета для видов бюджетов из УПП 1.3 и заполнять плановые данные.

Международный учет (только для перехода из УПП1.3 в УП2)

Переносится из УПП 1.3 план счетов Международный. В УП 2 план счетов Международный изначально не содержит предопределенных элементов. В УПП 1.3 план счетов содержит предопределенные элементы и может быть дополнен.

Субконто плана счетов Международный переносятся.

Остатки по счетам международного учета переносятся с использованием документа Операция (международный учет).

В итоге по международному учету пользователь получает привычный план счетов и начальное сальдо из УПП 1.3. Далее необходима настройка ведения международного учета в пользовательском режиме.

Зарплата и кадровый учет

Методика переноса данных зарплаты и кадрового учета совпадает с методикой переноса данных при переходе из редакции 2.5 в редакцию 3.0 конфигурации "1С:Зарплата и управление персоналом". Описание методики содержится в статье на сайте ИТС.

Перенос остатков

Основные условия и ограничения переноса

Основными данными для переноса считаются данные остатков счетов регламентированного учета. Там, где это возможно, переносятся остатки по управленческому учету. При этом превышение над остатками регламентированного учета записывается на Управленческую организацию.

При переносе действует несколько ограничений:

  • В случае, когда по данным управленческого учета количественные остатки больше, разница записывается на управленческую организацию. Таким образом после переноса остатки совпадут по регламентированному и управленческому учету. Если количественные остатки по регламентированному учету были больше, то произойдет коррекция данных управленческого учета - остатки по управленческому учету приравняются к регламентированному учету.
  • Отрицательные количественные или суммовые остатки по товарам не переносятся. Также не переносятся нулевые остатки.
  • После переноса остатков срабатывает проверка конечного сальдо на дату переноса остатков по счетам регламентированного. учета. В отчете о переносе данных можно ознакомиться с результатами проверки.
  • Остатки по счетам учета производства не переносятся.
  • На счетах учета товаров в информационной базе-источнике должен быть включен суммовой учет по виду субконто Склады. В противном случае не будет определен суммовой остаток товаров в разрезе складов, и товары не будут выгружены. После установки суммового учета по складам необходимо перепровести документы.

Материально-производственные запасы

В зависимости от учетной политики остатки получаются из различных регистров. При этом данные регламентированного учета и управленческого учета получаются из различных регистров и делятся на группы. В ячейках таблицы указаны названия правил выгрузки данных, по которым будет сопоставлена и выгружена данная группа данных. Названия правил выгрузки данных совпадают с названиями операций документа Ввод остатков.

Алгоритмы составления групп данных для регламентированного учета указаны в колонках, для управленческого учета - в строках. В колонке указывается пустое описание, когда данным управленческого учета не найдено соответствия в регламентированном учете. Такие данные выгружаются в остатки управленческой организации. Пустое описание в строке указывается тогда, когда данным регламентированного учета не обнаружено соответствия в данных управленческого учета. Такие остатки выгружаются в обычном порядке.

В зависимости от учетной политики способ переноса данных отличается:

Партионный учет

При партионном учете остатки по партиям товаров определяются по регистрам Партии товаров на складах (упр. учет), Партии товаров на складах (бух. учет) и НДС партии товаров (при сложном учете НДС).

Сопоставление производится по полям Организация, Склад, Номенклатура, Характеристика, Серия, Качество. Если ведется ФИФО, то к полям сопоставления добавляется Документ оприходования.

Данные регламентированного учета считаются основными при проведении сопоставления с данными управленческого учета. Все сопоставленные данные переносятся как остатки организаций по группам финансового учета, соответствующим счетам учета остатков МПЗ. Все несопоставленные данные в случае, когда количество по управленческому учету больше и в количестве разницы с партиями регламентированного учета, записываются как остатки по Управленческой организации. Если остатки управленческого учета в информационной базе-источнике были меньше остатков регламентированного учета, отрицательные количества по управленческому учету не записываются.

РАУЗ

При учетной политике РАУЗ возможна настройка аналитик, при которой в регламентированном учете не ведется учет характеристик, серий, качества. В этом случае данные переносятся без учета характеристик, серий, качества (сворачиваются).

Также при учетной политике РАУЗ возможна настройка ведения учета, при которой данные регистра Учет затрат (упр. учет) не содержат аналитику по организациям. В этом случае сопоставление остатков регламентированного и управленческого учета осуществляется по полям Номенклатура и Склад. В ситуации, когда остатки управленческого учета оказываются больше, чем регламентированного, излишек записывается как остатки Управленческой организации. В ситуации, когда остатки регламентированного учета оказываются больше, отрицательные значения на остатки Управленческой организации не записываются. Сопоставленные остатки по номенклатуре записываются в разрезе организаций и складов.

Стоимостная оценка остатков номенклатуры

Независимо от установленной учетной политики документ Ввод остатков по операции Остатки собственных товаров позволяет заполнять остатки в валюте регламентированного учета и управленческого. В алгоритме сопоставления партий сумма в валюте регламентированного учета берется из данных регламентированного учета. Сумма в валюте управленческого учета - из данных управленческого учета. Если количество МПЗ по управленческому учету отличается, то количество берется из данных регламентированного учета, а сумма в валюте управленческого учета берется пропорционально тому, на сколько больше или меньше количество по регламентированному учету.

Денежные средства

Для переноса остатков по банковским счетам и в кассам берутся остатки управленческого учета из регистра Денежные средства и остатки регламентированного учета по счетам 50, 51, 52.
Далее производится сопоставление остатков управленческого учета по остаткам регламентированного учета согласно следующим правилам:

  • Отрицательные остатки по кассам или расчетным счетам не переносятся.
  • В случае если остатки по счетам регламентированного учета больше, чем управленческого, на управленческую организацию ничего не переносится.
  • В остатках регламентированного учета нет данных о кассах и банковских счетах. В остатках управленческого учета нет данных о счетах учета. Поэтому в алгоритме производится сопоставление остатков для определения счетов учета и касс/банковских счетов.

Взаиморасчеты

Остатки по взаиморасчетам в регламентированном учете определяются по счетам 60, 62, 66, 67 регистра бухгалтерии Хозрасчетный. Данные получаются в разрезе договоров, организаций, контрагентов.

Проводится сопоставление остатков управленческого учета по остаткам регламентированного. Сопоставленные суммы записываются как остатки организаций, несопоставленные - по управленческой организации.

В случае если клиент заплатил предоплату, а счет учета указан неавансовый, то появляется отрицательное сальдо по кредиту указанного неавансового субсчета. В этом случае выводится сообщение об ошибке, т.к. авансы должны отражаться на авансовых субсчетах счетов учета расчетов. Данная сумма не переносится.

Для корректного зачета авансов необходимо запустить обработку Восстановление состояния расчетов.

Если собственная организация заплатила предоплату поставщику, и при этом указан неавансовый субсчет учета расчетов, то выводится сообщение об ошибке и данный остаток не переносится.

Подотчетные лица

Проводится сопоставление остатков управленческого учета по остаткам регламентированного. Сопоставленные суммы записываются как остатки задолженности подотчетных лиц в разрезе организаций.

Прочие активы и пассивы

Остатки переносятся в документ Ввод остатков с операцией Прочие активы и пассивы и документ Операция (регл. учет). В результате получаются корректные остатки по счетам регламентированного учета, а также остатки по статье управленческого баланса Прочие активы и пассивы.

Прочие счета

В данной группе переносятся остатки счетов, для которых в УП 2 / КА 2 методикой учета не предусмотрен ввод остатков. Перенос осуществляется только в документ Операция (регл. учет).

Внеоборотные активы

Используются регистры сведений и регистры накопления из информационной базы-источника для получения значений основных средств или нематериальных активов. Информация о принятых к учету основных средствах и нематериальных активах переносится с помощью документа Ввод остатков внеоборотных активов.
